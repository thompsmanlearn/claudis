## What Was Built

### lesson_injector (n8n workflow MFmk28ijs1wMig7h)
A 6-node sequential n8n workflow:
`Webhook → Extract Params → Query Lessons → Query Errors → Query Patterns → Assemble Context → Respond`

**Input:** `POST /webhook/inject-context` `{task_type, task_id, description}`
**Output:** `{context_block, lesson_count, queries}`

Calls `/memory_query` on the stats server (new endpoint added to stats_server.py) which proxies `query_texts` to ChromaDB using the MCP venv (avoiding the version mismatch between chromadb-client 1.5.5 and server 0.5.20).

### /memory_query endpoint (stats_server.py)
New FastAPI endpoint at `POST http://localhost:9100/memory_query`.
Accepts `{collection, query_text, n_results, distance_threshold}`.
Invokes MCP venv Python subprocess to run `collection.query(query_texts=[...])`.
Returns `{"results": [...], "count": N}` — wrapped to prevent n8n from unwrapping arrays into multiple items.

### scheduler.sh integration
Before every `claude -p` invocation:
1. Fetches top pending task from work_queue via Supabase REST
2. POSTs to lesson_injector webhook
3. Prepends returned CONTEXT BLOCK to wake_prompt via temp file
4. Passes enriched prompt to `claude -p` via stdin

Fails gracefully — if lesson_injector is unavailable, proceeds without enrichment.

## Bugs Encountered and Fixed

1. **`$helpers` not defined** in n8n Code node v2 sandbox — switched to native HTTP Request nodes
2. **JSON array unwrapping** — n8n HTTP Request node unwraps array responses into multiple items, breaking sequential chains. Fixed by wrapping response in `{"results": [...]}` at the stats server
3. **`specifyBody: "json"` + `jsonBody` expression** — correct format for HTTP Request node body (confirmed from TCA "Fetch Status" node pattern)
4. **`$json.body.field` vs `$json.field`** — webhook v2 wraps POST body in `.body`; Set node expressions need to handle both
5. **Collection IDs hardcoded** — ChromaDB REST API requires pre-computed embeddings for `query_texts`; solving via Python subprocess proxy was cleaner than trying to generate embeddings in JavaScript
