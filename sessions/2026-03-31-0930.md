## Task: Fix lesson_injector times_applied tracking

### Problem diagnosed
The lesson_injector (workflow MFmk28ijs1wMig7h) retrieved lessons from ChromaDB but never incremented `times_applied` in Supabase. Result: 68/93 lessons showed zero_applied despite 8.25 searches/task. The utilization metric was structurally broken — it measured searches but not whether lessons landed.

### Root cause discovered mid-build
ChromaDB and Supabase `lessons_learned` have divergent content — only **8.2% of lessons match** by content string. Same lesson gets written with different wording to each store (close-session generates summaries independently). Content-based matching would fail for 91.8% of lessons.

### Solution built
1. **Supabase RPC functions** (no management PAT needed, service key sufficient):
   - `increment_lessons_applied(lesson_contents text[])` — match by content (legacy fallback)
   - `increment_lessons_applied_by_id(lesson_ids text[])` — match by chromadb_id (primary)
2. **`chromadb_id` column** added to `lessons_learned` with index
3. **Backfill script** — keyword matching of ChromaDB ID slug against Supabase title (Jaccard ≥ 0.35, two passes with underscore-to-space fix) → **51/96 lessons** backfilled
4. **`/lessons_applied` endpoint** added to stats_server.py — accepts `{ids: [...], contents: [...]}`, tries by_id first, falls back to by_content
5. **lesson_injector workflow updated** — `Assemble Context` now outputs `lesson_ids` + `lesson_contents`; new `Update Times Applied` HTTP Request node calls stats_server before responding

## Decisions

- **RPC over direct SQL**: Supabase PostgREST can't do atomic increments via PATCH — RPC functions are the correct pattern. No management PAT needed.
- **chromadb_id as primary key** for matching: content diverges at 92% — the only reliable link between stores is a stored identifier. chromadb_id column solves this permanently going forward.
- **Two-pass backfill**: first pass missed underscored titles ("output_quality" → "outputquality"). Second pass with underscore→space normalization added 12 more matches.
- **continueOnFail on Update Times Applied node**: tracking failure must not block the context injection response.
