## Root Cause Analysis — AGS Failures

The AGS "Insert Work Item" node was missing `responseFormat: "text"` in its options. All other HTTP Request GET nodes in AADP have this setting; the POST node was created without it. When Supabase returns anything other than the expected JSON structure (edge cases with RLS-enabled tables post-2026-04-04 fix, empty responses), n8n throws "Empty or invalid json".

Additionally, `Prefer: return=representation` was unnecessary — the AGS never uses the returned row data. Changed to `Prefer: return=minimal`.

The 13:00 UTC "success" on 2026-04-05 was a false positive — the gate short-circuited because the queue had a pending task (manually inserted recovery), so the INSERT was never attempted.
