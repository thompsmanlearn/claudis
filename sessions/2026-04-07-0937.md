## Decisions

- `api_usage` is the correct column name for Anthropic API token usage data in `experimental_outputs` — semantic, queryable, not overloaded.
- n8n REST API PUT for workflow update rejects `active` as read-only (use `workflow_activate`/`workflow_deactivate` separately). MCP `workflow_update` handles this automatically.
- When MCP `workflow_update` works, prefer it. When the payload is too large for the MCP tool, use the n8n REST API directly with minimal fields: `{name, nodes, connections, settings, staticData}`.
