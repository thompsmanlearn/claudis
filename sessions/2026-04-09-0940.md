### 1. AGS Second-Round Fix — Real Root Cause Found and Applied

**Discovery:** The `sendBody: true` fix from last session (2026-04-09-0129) was saved to the workflow (confirmed via workflow_get, updatedAt 2026-04-09T01:37:38). But execution 2186 at 07:00 UTC still failed with "Empty or invalid json." The fix didn't fix the real problem.

**Root cause (layer 2):** `specifyBody: "raw"` + `body: "={{ $json.body }}"` where `$json.body` is a pre-stringified JSON string from an upstream Code node. Even with `sendBody: true`, this pattern produces a malformed body when sent to Supabase. The string is stringified once in the Code node (`JSON.stringify(insertPayload)`) and then n8n processes it again when evaluating the expression, resulting in corruption.

**Key insight:** Existing lesson `lesson_n8n_rawbody_complex_json_2026-03-30` (times_applied was 0 — never applied!) documented this exact pattern and specified the fix. That lesson was not injected when the AGS was originally built, and not applied during last session's debug. This session found it by searching before applying the fix.

**Fix attempt 1 (wrong):** Changed to `specifyBody: "json"` with object expression — but the existing lesson also says this fails with nested objects (like `input_data` for agent_build task which has arrays). Would work for explore/research_cycle but fail for agent_build.

**Fix attempt 2 (correct):** `specifyBody: "raw"` + inline `JSON.stringify` in body expression:
```
body: "={{ JSON.stringify({ task_type: $json.task_type, status: $json.status, priority: $json.priority, created_by: $json.created_by, input_data: $json.input_data }) }}"
```

Also changed `Build Task Payload` Code node to output fields as a proper object (not pre-stringified), so the Insert Work Item node can reference them directly.

**Verification:** Direct Supabase API call with the same pattern succeeded — INSERT returned the created row. Next AGS scheduled run is ~13:00 UTC; that will be the structural verification.
