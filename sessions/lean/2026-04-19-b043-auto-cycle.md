# Session: 2026-04-19 — B-043: Auto-cycle between sessions

## Directive
When a lean session completes successfully and the active project has remaining unblocked pending nodes, automatically trigger the next lean session without human intervention.

## What Changed

- **lean_runner.sh** (`~/aadp/sentinel/lean_runner.sh` + `~/aadp/claudis/sentinel/lean_runner.sh`) — Added AUTO-CYCLE block (~90 lines) at end of success branch. Logic:
  1. Checks `auto_cycle_enabled` in system_config — if false, skips (default)
  2. Queries active projects from `aadp_projects` (status=active)
  3. Fetches all nodes for those projects; finds first pending node with all dependencies done
  4. Writes the node as a full directive to `DIRECTIVES.md` (name, goal, context, node ID)
  5. Commits and pushes DIRECTIVES.md to claudis repo
  6. Releases the session lock, waits 2s, calls `localhost:9100/trigger_lean`
  7. Sends Telegram notification with node name
  8. If no pending nodes remain, marks active projects status='complete'

- **system_config** — Added `auto_cycle_enabled = false`. Bill enables this from Anvil dashboard when he wants sessions to chain automatically.

- **claudis/sentinel/lean_runner.sh** — committed to git (was disk-only, now version-controlled). Push: commit `51fdc1a`.

## What Was Learned

- lean_runner.sh was already in the claudis repo (as a modified tracked file) — the "disk-only" description in TRAJECTORY.md was slightly stale.
- Writing DIRECTIVES.md via `python3 -c` rather than a bash heredoc avoids variable-expansion issues when node content contains `$` characters.
- The lock file must be explicitly removed (and the trap reset) before calling trigger_lean — otherwise stats_server sees the current session's lock as still active and rejects the spawn.
- B-042 was already complete when this session started — DIRECTIVES.md was stale. Root cause: the close-session ritual for B-042 didn't update DIRECTIVES.md to the next card. Future: include DIRECTIVES.md update in close-session or have lean_runner.sh update it after cycle completes.
