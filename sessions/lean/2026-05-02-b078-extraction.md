## Capability Delta

**New:** Anvil can now accept desktop-Claude analysis pastes and recover structured implications — synthesis, conclusions, screening decisions, and candidate sub-questions. Confident screening decisions commit to research_articles immediately; uncertain ones surface in the thread with inline resolution controls.

**Design decisions made:**
- Haiku 4.5 — no cache_control per DEEP_DIVE_BRIEF Section 12
- `extract_analysis` extracts only; Form1 writes entries (separate concerns)
- `_make_screening_handlers` factory pattern avoids closure-in-loop Python issue
- `screening_uncertain` entry content is JSON; metadata.resolved flags post-resolution
- PATCH to research_articles in `resolve_screening_uncertain` replaces both rating and status atomically

## What I Learned

- `_make_screening_handlers` factory pattern is the right idiom for Anvil action closures that need per-entry state — cleaner than default-argument capture in loop body
- Supabase JSONB PATCH via PostgREST accepts Python dicts directly (no json.dumps needed)
- anthropic 0.97.0 installs cleanly into the mcp-server venv
