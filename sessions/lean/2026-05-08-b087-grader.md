## What Was Learned
The HEAD~3 git diff window is an inherent limitation: if a card was completed more than 3 commits ago, the grader won't see its changes in the diff. In a production auto-cycle, the graded card is always the most recent, so HEAD~3 usually captures it. But for manual smoke tests on older cards, the grader will always pause due to missing git evidence. Future improvement: pass the specific commit SHA to the grader endpoint.

The grader correctly does not fabricate evidence — it issues "pause" rather than "pass" when git confirmation is absent. This is the right behavior.
