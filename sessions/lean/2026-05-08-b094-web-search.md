## What Was Learned
Initial web_fetch read only `max_chars * 4` bytes of raw HTML, which only captured `<head>` content on most pages (80–185 chars visible after stripping). Fixed by reading up to 200KB before stripping — now returns real body content. JavaScript-rendered SPAs will still return minimal content since we don't execute JS; this is acceptable for research use cases (academic papers, documentation, news articles are server-rendered).

Some sites (raspberrypi.com) return HTTP 403 for non-browser user agents even with robots.txt allowed. This is expected — web_fetch degrades gracefully with error response, not exception.
